# Figure-audit ‚Äî catch AI hallucinations in schematic figures before submission

## When to invoke

Use this skill whenever the user says any of:
- "audit my figures" / "check the figures"
- "any text artefacts" / "any AI hallucinations"
- "scan the Extended Data figures"
- "figure artefact check before submission"
- "are the figures clean"
- "before I export the figures to TIFF"

Or proactively, **before any submission** where the figure list includes Gemini/DALL-E/Midjourney/SD generation:
- File names matching `Gemini_Generated_*`, `DALLE_*`, `MJ_*`, `SD_*`, `firefly_*`
- Figures described in the manuscript as "schematic", "illustrative", "graphical abstract", "Extended Data Fig"
- Figures whose corresponding `.R` or `.py` ggplot script does NOT exist in the analysis repo (suggests AI source)

## The five recurring AI-figure artefact classes

Based on observed failures in the perimenopause / FH / CALON projects:

| Class | Example | Detection signal |
|---|---|---|
| **Latin-alphabet hallucination** | `VLDL lLopiite` (should be `droplet`), `abundatl` (should be `abundant`) | Word that "looks like" English but is not in any dictionary |
| **Impossible anatomy** | Vein labelled on an artery; LDL receptor on RER (it lives on plasma membrane); mitochondrion drawn with two outer membranes | Anatomical reviewer needed |
| **Unit-formatting errors** | `mmol\/L`, `mmol/||`, `nmol¬∑L‚Åª¬π` rendered as `nmol¬∑L"¬π`, `Œî` rendered as box | Glyph cannot be reproduced as plain text |
| **Mirrored / rotated letters** | Reversed `R` in middle of word; rotated 6/9 confusion in a numeric label | Pattern doesn't repeat consistently elsewhere |
| **Phantom arrows / orphan labels** | Arrow that ends in the middle of empty space; label with no anchor; legend that mentions a colour not present in the figure | Topological inconsistency |

## Five-step protocol

### Step 1 ‚Äî Identify the AI-generated subset

Scan the figures folder and classify each file:

```python
from pathlib import Path

AI_NAME_PATTERNS = [
    'Gemini_Generated', 'firefly_', 'Adobe_Firefly',
    'DALLE_', 'dalle_', 'OpenAI_Generated',
    'Midjourney_', 'MJ_', 'mj_',
    'StableDiffusion_', 'SD_', 'sd_',
    'Google_AI_Studio', 'Stability_',
]

def is_likely_ai_generated(path: Path) -> bool:
    name = path.name
    if any(p in name for p in AI_NAME_PATTERNS):
        return True
    # Heuristic: matched ggplot scripts exist? if not, AI is likely
    stem = path.stem
    has_script = any(
        Path(path.parent.parent / 'scripts' / f'{stem}.{ext}').exists()
        for ext in ('R', 'py', 'cxc')
    )
    return not has_script
```

Save the AI-subset as `figure_audit/ai_figures.csv` with columns `path, size, likely_artefact_risk`.

### Step 2 ‚Äî Visual inspection prompts for each figure

For each AI figure, **read the image with the Read tool** (Claude can see PNG/JPG). Then in the same conversation turn, run this checklist:

```
For figure X (panel(s) Y):
1. Read every label out loud. Does each word exist in English? Flag: nonsense Latin-alphabet strings (e.g. "lLopiite", "abundatl").
2. Trace every arrow from tail to head. Does each arrow point at a labelled object? Flag: phantom arrows.
3. Compare the legend caption to what is rendered. Are all colours, shapes, labels in the legend present in the panel? Flag: orphan legend entries.
4. Verify units. Are all units rendered as expected ASCII (mmol/L, nmol/L, mg/dL)? Flag: corrupted unit glyphs.
5. Verify anatomy. Are organ relationships and cellular locations biologically correct? Flag: vein-as-artery, wrong organelle, wrong membrane.
```

Record findings in `figure_audit/findings.csv`:
- `figure`, `panel`, `region` (e.g. "upper-left, x‚âà0.18 y‚âà0.07"), `artefact_class`, `current_text`, `correct_text`, `severity` (BLOCKER / WARNING / INFO).

### Step 3 ‚Äî Decide the correction approach

Three correction strategies in order of safety:

1. **Vector overwrite** (preferred) ‚Äî open in Illustrator / Photopea, eyedropper the background, draw a filled rectangle over the artefact, retype with the correct font. Preserves the rest of the figure perfectly.
2. **PIL coordinate overwrite** ‚Äî for batch / automated fixes when ~5 figures need the same artefact removed. Less pixel-perfect; always do a final visual check.
3. **Full regeneration** ‚Äî last resort. Only if the artefact is structural (impossible anatomy). Document the prompt and verify the new figure does not introduce different artefacts.

**Never** simply ask the original AI model to "regenerate without the artefact" without changing the prompt ‚Äî image models often introduce new, different artefacts on each generation.

### Step 4 ‚Äî PIL coordinate-overwrite template

For class 1 (Latin-alphabet) and class 3 (unit-formatting) where the region is well-localised:

```python
from PIL import Image, ImageDraw, ImageFont
from pathlib import Path

def patch_figure(src: Path, dst: Path,
                 region: tuple[float, float, float, float],  # (x0, y0, x1, y1) in [0,1]
                 background_rgb: tuple[int, int, int],
                 corrected_text: list[str],
                 font_size_frac: float = 0.025):
    """Overwrite a labelled region with a colour-matched box + corrected text."""
    img = Image.open(src).convert('RGB')
    w, h = img.size
    draw = ImageDraw.Draw(img)

    x0, y0, x1, y1 = (int(w * region[0]), int(h * region[1]),
                       int(w * region[2]), int(h * region[3]))
    draw.rectangle([x0, y0, x1, y1], fill=background_rgb)

    try:
        font = ImageFont.truetype("arial.ttf", int(h * font_size_frac))
    except Exception:
        font = ImageFont.load_default()

    for i, line in enumerate(corrected_text):
        draw.text((x0 + 8, y0 + 4 + i * int(h * font_size_frac * 1.2)),
                  line, fill=(0, 0, 0), font=font)

    dst.parent.mkdir(parents=True, exist_ok=True)
    img.save(dst, 'PNG', dpi=(600, 600))
```

### Step 5 ‚Äî Final visual sign-off

After patching, **read the patched image with the Read tool again** and confirm:
- The artefact is no longer visible
- The corrected text is positioned correctly (not overlapping other labels)
- The font and size match the rest of the figure
- The image still saves at ‚â•300 dpi (TIFF or PNG)

Do NOT mark the audit complete until this visual sign-off passes.

## ICMJE AI-figure disclosure requirement

Whenever AI-generated figures appear in a manuscript, the AI declaration MUST disclose:
- The specific AI tool and version (e.g. "Google Gemini 2.0", "DALL-E 3", "Midjourney v6")
- The date range of use
- That all embedded text has been manually verified and corrected
- That no patient data appears in the AI-generated figures
- That analytical figures (ggplot, matplotlib) are AI-free

The `manuscript-qc` skill enforces this disclosure as part of its sign-off checklist.

## Reference exemplar

**The "VLDL lLopiite" incident (April 2026, perimenopause manuscript):**

A Gemini-generated dual-engine schematic for the menopausal lipid remodelling paper contained the text `VLDL lLopiite` inside the yellow vesicle cluster ‚Äî an obvious AI hallucination of the word `droplet`. A second Gemini figure (`Gemini_Generated_Image_hixi2dhixi2dhixi.png`) contained `abundati` (should be `abundant`).

Both artefacts went undetected through:
- The original draft round
- The first reviewer-style mock review
- The initial QC pass

They were caught only by direct PIL.Image.read inspection during the final QC pass. The corrected figures were patched using the PIL coordinate-overwrite pattern (`patched/Gemini_ED3_dual_engine_FIXED.png`, `patched/Gemini_ED1_abundant_receptors_FIXED.png`).

**Lesson encoded in this skill:** *Always read every AI-generated figure with the Read tool, label-by-label, before declaring a manuscript submission-ready.*

## Pre-flight environment check

Before patching:
```bash
python -c "from PIL import Image, ImageDraw, ImageFont; print('OK')" || pip install Pillow
```

Ensure `arial.ttf` or another sans-serif font is available on Windows (System32/Fonts). Fall back to `ImageFont.load_default()` if not.

## Common pitfalls

- **Patched text is anti-aliased differently from the original** ‚Äî the patch looks "stuck on" rather than integrated. Mitigation: use the same PIL antialias settings as the source; for journal export, this difference is rarely visible at print resolution.
- **Eyedropper background colour is wrong** ‚Äî auto-sampled colour doesn't match because the original has subtle gradient. Mitigation: sample several pixels and average; or use Illustrator instead of PIL for gradient regions.
- **Font not found** ‚Äî `arial.ttf` is on Windows but not Linux. Always wrap in try/except and fall back to `load_default()`.
- **Patched figure looks worse than the artefact at thumbnail scale** ‚Äî verify at the actual figure size that will appear in print (typically 8 cm or 17 cm width), not at zoom 1:1.
- **Forgot to update the AI declaration** ‚Äî every AI-figure patch round triggers a re-check of the manuscript's AI declaration statement (see `manuscript-qc` Part G template).
